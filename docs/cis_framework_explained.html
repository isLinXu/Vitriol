CIS 压缩智能评分框架

用一把「四维尺子」衡量模型压缩策略的聪明程度
不只是压得更小——压缩即智能

Ψ(S) = α·ηinfo + β·ηstorage + γ·ηexpress + δ·Ttrain
💡 一句话先说清楚
想象你要把一套《大英百科全书》塞进一个背包。

有人直接撕掉一半页（❌ 信息丢了）→ 有人用缩印字体（✅ 字小但还能读） → 还有人提炼成「知识图谱」（🌟 最高级——不仅省空间，还让知识结构更清晰）。

CIS 框架就是那个「评委」——它从 4 个维度给每种「打包方法」打分，最后得出一个综合分数 Ψ（读作 Psi）。
核心概念

四个维度，像四只眼睛看同一个问题

CIS 不是单一指标，而是从四个不同角度综合评价一个压缩策略的「智商」

ηinfo

信息保留度

压缩后，原始模型的「记忆」还剩多少？
如果把模型比作一个人，这个分数回答：「他失忆了吗？」
🔬 用 SVD 分解比较压缩前后权重矩阵的奇异值分布差异 → 越接近 = 失忆越少

📊 辅助：熵分析、频谱相关系数
ηstorage

存储效率

到底压缩了多少？70B 模型变 1MB 是本事，变 69GB 就没意义。
回答：「省了多少空间？」
📦 压缩率取 对数映射到 0~1 分
  99%压缩 → 接近满分 | 0%压缩 → 0分

🧮 额外考量：稀疏度、元数据开销
ηexpress

表达能力

压缩后的权重还有多「聪明」？全零矩阵极度节省空间但毫无用处。
回答：「它还能表达复杂模式吗？」
🎨 三管齐下判断权重的「丰富程度」：
  ① 值多样性：有多少不同的数值？
  ② 分布复杂度：偏度、峰态、非零比例
  ③ 矩阵秩：有效秩 / 理论满秩
Ttrain

可训练性

压缩后还能继续微调训练吗？有些方法把权重「杀死了」，梯度传不回去。
回答：「它还有学习潜力吗？」
⚡ 三项体检：
  ① 条件数分析：梯度流是否通畅？
     条件数 <100 → 好 | >10000 → 梯度消失/爆炸风险
  ② 信号尺度：权重值域是否合理？
  ③ 方差保持：与原始权重的方差比值是否在健康范围？
数学本质

公式拆解：每个符号都是一句人话

别被希腊字母吓到——本质上就是一个加权平均

🎯 Ψ(S) — PSI 分数（最终得分）
范围 0 ~ 1，越高 = 压缩策略越「聪明」。这是四个维度的加权总和。
α = 0.30 — 信息保留的权重（最高优先级之一）
再怎么压缩，不能把模型的核心知识弄丢。这是底线。没有信息的压缩就是破坏。
β = 0.30 — 存储效率的权重（同样重要）
压缩的目的就是省空间。不省空间那叫什么压缩？但光省空间不保留信息也不行。
γ = 0.25 — 表达能力的权重
省了空间但如果权重退化成一堆常数或全零矩阵，模型就「变傻」了。表达能力保证模型还能做复杂的推理。
δ = 0.15 — 可训练性的权重
如果你打算后续微调（fine-tune），这个维度很关键。纯推理部署场景下可以适当降低。
⚖️
为什么 α 和 β 相等且最高？
因为 CIS 的核心理念是「在保留信息和压缩体积之间找最佳平衡点」。这俩是矛盾的两极——信息保留越多通常意味着压缩越少。必须同时优化才能找到最优解。

🔬 维度一：信息保留度 ηinfo — SVD + KL 散度的故事 ▶

核心思想

任何一个权重矩阵都可以用 SVD（奇异值分解）分解成三个部分的乘积：

W = U · Σ · VT

其中 Σ 对角线上的数字叫「奇异值」——它们代表了矩阵在不同方向上的「能量大小」。

🎵 类比：音频均衡器
把一首歌做傅里叶变换，你会看到不同频率的能量分布（低音、中音、高音各占多少能量）。

SVD 对矩阵做的事情类似——它告诉你这个权重矩阵中，「重要程度」不同的方向各有多少能量。大的奇异值 = 重要的特征方向。

计算步骤（3 步）

  1. 对原始权重 Worig 做 SVD → 得到奇异值向量 Sorig
  2. 对压缩后权重 Wcomp 做 SVD → 得到奇异值向量 Scomp
  3. 将两者归一化成概率分布，计算 KL 散度衡量差异

为什么用 KL 散度？

KL 散度衡量的是「两个概率分布有多不一样」：

KL(P||Q) = Σ P(x) · log( P(x) / Q(x) )
  • 如果压缩前后的奇异值分布完全一样 → KL = 0 → 信息完全保留 → 得分 = 1.0
  • 如果分布差很远 → KL 很大 → exp(-KL) 趋近于 0 → 得分 ≈ 0

辅助指标

熵分析权重值的分布熵——熵越大说明信息量越丰富
频谱保持FFT 后的功率谱相关性——频率域的信息丢失检测
📦 维度二：存储效率 ηstorage — 对数尺子的智慧 ▶

直觉陷阱

如果你直接用 1 - compressed/original 来打分，会遇到一个问题：

方案 A：压缩到 50%
得分 = 0.50
VS
方案 B：压缩到 1%
得分 = 0.99

看起来还行？但从 50% 到 1%，工程难度差了 几个数量级！线性评分无法体现这种差异。

CIS 的解决方案：对数尺度

score = -log10(ratio) / 2

// 示例：
// ratio = 0.50 (50%) → score = -log(0.5)/2 = 0.15
// ratio = 0.01 (1%) → score = -log(0.01)/2 = 1.00 ✅
// ratio = 0.0001 → score = -log(0.0001)/2 = 2.00 → clamp 到 1.0
💡 为什么用对数？
因为压缩比的改善是指数级别的。从 90% 压缩到 99% 看起来只多了 9%，但实际上体积又缩小了 10 倍！对数尺度才能公正地评价这种成就。

额外考量因素

  • 稀疏度(sparsity)：权重中有多少是零？零越多 = 实际存储越小
  • 元数据开销(metadata overhead)：有些压缩方法需要存索引表、码本等，这些也算成本
🎨 维度三：表达能力 ηexpress — 不是越小越好！ ▶

一个反直觉的事实

极致的压缩可能让权重变得毫无用处。

❌ 全零矩阵
存储效率: 满分 (≈0字节)
表达能力: 0分
模型输出: 全是垃圾
VS
✅ 结构化稀疏
存储效率: 良好 (80%)
表达能力: 优秀 (0.80)
模型输出: 可用的推理结果

三项子指标的详细解释

① 值多样性 (Value Diversity)

统计权重中有多少个不同的数值。

unique_count / total_elements

// 全部相同 → diversity = 1/N ≈ 0 （极差）
// 全部不同 → diversity = 1.0 （极好）

⚠️ 但注意：二值化权重只有 2 个值，diversity 极低却不一定差——所以需要结合其他指标！

② 分布复杂度 (Distribution Complexity)

用统计学特征描述权重分布的「形状」：

偏度 (Skewness)分布是否偏向一边？对称分布偏度=0，偏度大说明有方向偏好
峰度 (Kurtosis)分布是尖锐还是平坦？高峰度 = 少数极端值占主导
非零比例实际参与计算的权重占比

三者加权融合得到复杂度分数。

③ 矩阵秩 (Rank Score)

通过 SVD 计算矩阵的有效秩：

// 统计 "显著大于零" 的奇异值个数
threshold = max_singular_value × 1e-5
effective_rank = count(singular_values > threshold)
rank_score = effective_rank / min(rows, cols)
🏗️ 工程类比
想象一个建筑工程团队。如果所有人都会一样的技能（低多样性）、都按同一模式工作（低复杂度）、只能做一种类型的活（低秩），那这个团队再大也没用。

好的权重就像一个技能多样的专业团队——每个人有不同的专长，组合在一起能解决各种复杂问题。

最终融合

η_express = 0.40×diversity + 0.30×complexity + 0.30×rank_score
⚡ 维度四：可训练性 Ttrain — 权重还「活」着吗？ ▶

为什么需要这个维度？

很多压缩方法只关注「压缩后能不能正常推理」，却忽略了另一个关键问题：压缩后的权重还能继续学习吗？

🔥
真实场景：你拿到一个 70B 的开源模型，先用 Ultra 策略压缩到 164KB，然后想在你的私有数据上微调。如果压缩过程「杀死了」梯度流，微调就完全不可能了！

三项体检指标

① 梯度流分数 (Gradient Flow Score)

核心思想：通过 SVD 的条件数(Condition Number)来判断梯度是否能顺利传播：

cond_number = σ_max / σ_min // 最大奇异值 / 最小奇异值

// 条件数的含义：
// cond ≈ 1 → 矩阵「健康」，梯度完美传播 → score = 1.0
// cond = 100 → 轻微病态，梯度略有衰减 → score ≈ 0.77
// cond = 10000+ → 严重病态！梯度消失/爆炸 → score ≈ 0.26

score = 1.0 / (1.0 + log10(cond + 1))
📻 无线电类比
条件数就像无线电的信噪比。信噪比好时，信号清晰传输；信噪比差时，信号要么被淹没（梯度消失），要么失真爆炸（梯度爆炸）。

好的压缩应该尽量保持条件数在一个合理范围内（< 100~1000）。

② 信号尺度分数 (Signal Scale Score)

检查权重的绝对值范围是否在「训练友好区」：

mean_abs = mean(|W|)

if mean_abs < 1e-6: // 太小 → 梯度也小 → 学习停滞
  score = mean_abs / 1e-6
elif mean_abs > 1.0: // 太大 → 可能数值不稳定
  score = 1.0 / mean_abs
else: // 在 sweet spot (1e-6 ~ 1.0)
  score = 1.0 // ✓ 最佳区间

③ 方差保持 (Variance Preservation)

基于 Xavier/He 初始化理论——好的权重应该在各层间保持方差稳定：

ratio = var(W_compressed) / var(W_original)

// ratio ≈ 1.0 → 方差保持良好 → score 高
// ratio 偏离太远 → 违反初始化理论 → score 低

// 健康范围: 0.5 ≤ ratio ≤ 2.0

最终融合

T_train = 0.40×gradient_flow + 0.30×signal_scale + 0.30×variance_preservation

最迷人的发现

CIS 框架最独特的贡献不是那四个分数，而是发现了一个相变现象：

🌋 当压缩率跨过 ~90% 时，会发生质变

这就像水在 0°C 从液体变成固体一样——性质发生了根本变化。

两种「智能状态」

📦 压缩率 < 90%：存储模式
• 本质还是在「打包信息」
• 压掉的是冗余和噪声
• 核心知识结构完整保留
• 类比：把书缩印
🧠 压缩率 ≥ 90%：抽象模式
• 不再是「打包」而是提炼本质
• 只保留最核心的规律
• 大量细节丢失但骨架清晰
• 类比：把书写成摘要

如何探测临界点？

CIS 使用二阶导数（加速度）来检测表达能力的突变：

// 想象你在开车：
// 速度在慢慢减 → 一阶导数为负（正常减速）
// 突然猛踩刹车 → 二阶导数很大（加速度突变！）

// 同理，当「表达能力下降的速度」突然加速时，
// 说明我们跨过了某个临界点

second_derivative = Δ(expressivity_loss_rate) / Δ(compression_ratio)
critical_point = argmax(|second_derivative|)

📌 这个临界点不是固定的 90%，而是根据实际数据动态检测出来的。90% 只是经验观察值。
